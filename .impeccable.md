## Design Context

### Users
Primary users are developers debugging backend request paths in local and staging environments.
They need to quickly understand where a request failed, where latency accumulated, and which layer produced the issue.

### Brand Personality
Technical, precise, premium.
The interface should evoke confidence and control, with a focused and understated tone.

### Aesthetic Direction
Dark-mode-first, matte surfaces, restrained accents, dense but readable information.
Reference direction: Linear / Vercel / Datadog clarity without noisy observability clutter.
Anti-reference direction: playful gradients, glassmorphism-heavy cards, oversized rounded dashboard templates.

### Design Principles
1. Prioritize trace readability over decoration.
2. Maintain clear hierarchy with compact, developer-friendly density.
3. Use color semantically (success/error/slow) and sparingly.
4. Keep interactions fast, keyboard-friendly, and predictable.
5. Build modular UI and data transformations for scale.
